Giai thich Use Case Diagram

He thong SAINT su dung mo hinh RBAC 2 vai tro giong mo hinh quan ly phong hoc (Admin = Quan tri vien, Teacher = Giao vien). Moi actor co pham vi quyen rieng biet:

Admin (Quan tri vien)
Toan quyen he thong. Tao/xoa tai khoan Teacher, quan ly tat ca Groups, Agents, Whitelist, API Keys, xem Audit Log, cau hinh he thong. Chi co 1 tai khoan Admin, tao san khi deploy. Admin co the thao tac tren tat ca Groups cua moi Teacher.
Teacher (Giao vien)
Quan ly Group cua chinh minh. Teacher co the tao Group (lop hoc), them/xoa Agents, quan ly Whitelist rules trong Group do. Teacher chi thay va thao tac tren cac Group ma minh tao ra. Co the doi mat khau ca nhan, xem Dashboard va Logs cua Group minh.
Agent (Thiet bi)
Chuong trinh chay tren may client (Windows). Dang ky voi Server, gui heartbeat, gui log, nhan whitelist. Xac thuc bang API Key + JWT. Khong thay doi so voi he thong hien tai.
Nguyen tac thiet ke quan trong: Teacher chi co quyen CRUD tren cac Group (lop hoc) ma chinh minh tao ra (truong created_by). Teacher khong the thay hoac thao tac tren Group cua Teacher khac. Admin co the thay va quan ly tat ca Groups. Dieu nay dam bao data isolation giua cac Teacher.

Giai thich ER Diagram

He thong su dung MongoDB (document-based). Cac collection duoc thiet ke voi quan he ro rang:

  • USERS (MOI) — Collection trung tam cho RBAC. Luu tai khoan Admin va Teacher. Truong role la enum string (admin hoac teacher). Co che chong brute-force qua failed_login_attempts + locked_until. Truong created_by lien ket toi User da tao (Admin tao Teacher).
  • SESSIONS (MOI) — Moi lan dang nhap Admin/Teacher tao session. Luu JTI cua access_token va refresh_token de ho tro thu hoi (revoke). TTL index tu xoa session het han.
  • AUDIT_LOGS (MOI) — Ghi lai moi hanh dong thay doi du lieu. Truong details luu JSON diff (before/after). Ghi nhan ca role cua nguoi thuc hien de audit.
  • GROUPS (SUA) — Them truong created_by de xac dinh Teacher nao so huu Group. Day la truong quan trong nhat cho viec phan quyen — Teacher chi thay Groups co created_by = user_id cua minh.
  • WHITELIST_RULES (GIU NGUYEN) — Them created_by de tracking. Teacher chi quan ly rules trong Group cua minh.
  • AGENTS, API_KEYS, REVOKED_TOKENS (GIU NGUYEN) — Khong thay doi logic hien tai. Chi them created_by cho API_KEYS.
MongoDB Optimization: Index compound tren {username: 1}, {email: 1}, {role: 1, is_active: 1} cho USERS. Index {created_by: 1} tren GROUPS de query nhanh "tat ca Groups cua Teacher X". TTL index tren SESSIONS {expires_at: 1} va REVOKED_TOKENS {expires_at: 1}.

Giai thich Class Diagram

Thiet ke theo kien truc MVC + Service Layer, tuong thich hoan toan voi cau truc Flask hien tai cua SAINT:

  • UserModel (MOI) — MongoDB operations cho collection users. Tuong tu cac model hien co (AgentModel, GroupModel...). Cac method: CRUD, lock account, failed attempts tracking.
  • AdminAuthService (MOI) — Xu ly toan bo logic dang nhap Admin/Teacher. Tach biet hoan toan voi auth Agent hien tai (auth_controller.py). Password hash bang bcrypt, brute-force protection, session management.
  • RBACService (MOI) — Logic phan quyen thuan tuy. Method quan trong nhat: can_access_group(user, group) — kiem tra Teacher co phai owner cua Group khong. Admin luon tra ve true.
  • RBACMiddleware (MOI) — Tich hop voi Flask decorator pattern. @require_login() kiem tra cookie JWT. @require_admin() chi cho Admin. Hoat dong song song voi middleware auth.py hien tai (cho Agent API).
  • AdminAuthController (MOI) — Blueprint moi, prefix /admin/auth/*. Khong xung dot voi routes hien tai (/api/auth/* cho agents).
  • UserManagementController (MOI) — Chi Admin truy cap. CRUD tai khoan Teacher.
  • DashboardController (SUA) — Them @require_login() vao cac route dashboard hien tai + tao route /login moi. Teacher thay Dashboard gioi han theo Groups cua minh.
  • JWTService (TAI SU DUNG) — Service JWT hien tai duoc dung chung. Them payload fields user_id, role, username cho admin tokens.
  • AuditService (MOI) — Ghi log moi hanh dong thay doi. Format: user.create, group.update, whitelist.delete...

Giai thich Luong Dang Nhap

  • Buoc 1-2: User (Admin hoac Teacher) truy cap /login, server render trang login bang Jinja2.
  • Buoc 3-5: Form submit POST toi /admin/auth/login. AdminAuthService tim user trong MongoDB.
  • Buoc 6: Kiem tra account lock — neu da sai mat khau 5 lan lien tiep, tai khoan bi khoa 15 phut (chong brute-force).
  • Buoc 7: So sanh password bang bcrypt (constant-time comparison, chong timing attack).
  • Buoc 8-9: Tao cap JWT tokens. Access token chua {user_id, role, username} dung cho authorization. Refresh token chi chua {user_id}.
  • Buoc 10: Tao session record trong DB de ho tro: xem sessions dang active, revoke tu xa, audit trail.
  • Buoc 11: Ghi audit log. Moi lan dang nhap thanh cong/that bai deu duoc ghi lai.
  • Buoc 12: Token duoc luu trong httpOnly cookie (khong phai localStorage) — chong XSS attack. Sau do redirect toi Dashboard.
Luu y: Luong dang nhap nay hoan toan tach biet voi luong auth cua Agent (/api/auth/*). Agent van su dung API Key + JWT nhu hien tai. Admin/Teacher dung cookie-based JWT moi.

Giai thich Luong Authorization

Moi request toi protected route deu di qua 3 lop kiem tra:

  1. Authentication (Lop 1) — Co token hop le khong? Token co bi revoke khong? Neu het han, tu dong refresh.
  2. Permission Check (Lop 2) — User co quyen whitelist:create khong? RBACService kiem tra role → permissions mapping.
  3. Ownership Check (Lop 3 — DAC BIET CHO TEACHER) — Teacher co phai owner cua Group nay khong? Kiem tra group.created_by == user.user_id. Admin bo qua buoc nay (toan quyen).

Day la diem khac biet quan trong nhat: Teacher co quyen whitelist:create (tao rule), nhung chi duoc tao rule trong Group cua chinh minh. Admin co the tao rule trong bat ky Group nao.

3 lop bao ve: Authentication → Permission → Ownership. Request phai pass ca 3 lop moi duoc xu ly. Day la co che RBAC + Resource Ownership, dam bao Teacher chi thao tac tren du lieu cua minh.

Giai thich Activity Diagram

Day la luong tong quat cua moi HTTP request trong he thong SAINT sau khi tich hop RBAC. Diem quan trong:

  • Tach biet 2 luong auth: Agent API (/api/*) van dung API Key + JWT hien tai — khong thay doi gi. Admin/Teacher Web (dashboard routes) dung cookie-based JWT moi.
  • Auto-refresh token: Khi access token het han, he thong tu dong thu refresh bang refresh token. User khong can dang nhap lai trong 7 ngay.
  • 3 lop bao ve cho Teacher: Authentication → Permission Check → Ownership Check. Buoc Ownership la diem dac biet: Teacher phai la owner cua resource (Group) moi duoc thao tac.
  • Admin bypass Ownership: Admin luon pass buoc Ownership check — co quyen thao tac tren moi resource.
  • Audit logging: Moi request thanh cong den protected route deu ghi audit log (ai, lam gi, luc nao, tu IP nao).

Giai thich State Diagram

  • Unauthenticated: Trang thai mac dinh. Moi request toi protected route deu redirect ve /login.
  • Authenticating: Trang thai chuyen tiep khi user submit form. Server xac minh credentials.
  • Authenticated → ActiveSession: User dang online, access token con han. Moi request deu kiem tra token validity.
  • TokenExpired: Access token het han (24h). Browser tu gui refresh token de lay access token moi — transparent cho user.
  • SessionExpired: Ca refresh token cung het han (7 ngay). User phai dang nhap lai.
  • AccountLocked: Sau 5 lan dang nhap sai lien tiep → khoa 15 phut. Bao ve chong brute-force attack.
  • Force logout: Co 3 cach ket thuc session som: user tu logout, Admin revoke tu xa, hoac tai khoan bi deactivate boi Admin.

Permission Matrix — Ma tran quyen chi tiet

Ma tran phan quyen day du — Admin vs Teacher

Bang duoi day mo ta chi tiet quyen cua tung role doi voi moi resource va action. Luu y cot "Ownership" — cho biet Teacher co bi gioi han boi ownership khong.

Giai thich Permission Matrix

  • ✓ = Co quyen day du — Thao tac tren tat ca resources.
  • ~ Own = Co quyen gioi han boi Ownership — Chi thao tac tren resources ma Teacher so huu (created_by = user_id). Day la diem then chot cua thiet ke.
  • ✗ = Khong co quyen — Khong the truy cap chuc nang nay.
  • Format permission: resource:action — de mo rong, de kiem tra. Vi du: whitelist:create = quyen tao whitelist rule.
  • Nguyen tac: Admin co toan quyen (30 permissions). Teacher co 15 permissions nhung bi gioi han boi ownership tren Groups/Whitelist/Agents.

Implementation Code — Permission Config

# server/config/rbac_config.py ROLE_HIERARCHY = { "admin": 100, # Toan quyen "teacher": 50, # Gioi han boi ownership } ROLE_PERMISSIONS = { "teacher": [ # Profile "profile:read", "profile:change_password", # Dashboard "dashboard:read", # Groups (gioi han boi ownership) "groups:read", "groups:create", "groups:update", "groups:delete", "groups:manage_agents", # Agents (chi xem trong Group cua minh) "agents:read", "agents:detail", # Whitelist (gioi han boi ownership) "whitelist:read", "whitelist:create", "whitelist:update", "whitelist:delete", "whitelist:sync", # Logs (chi thay logs tu Group cua minh) "logs:read", ], "admin": [ # Ke thua toan bo quyen cua teacher + # User management "users:create", "users:read", "users:update", "users:delete", "users:reset_password", # Agent management day du "agents:delete", "agents:command", # API Keys "api_keys:read", "api_keys:create", "api_keys:revoke", # Logs day du "logs:export", "logs:delete", # System "system:config", "audit:read", ], } def get_all_permissions(role: str) -> list: """Get all permissions for a role including inherited ones.""" if role == "admin": # Admin ke thua toan bo quyen cua teacher return list(set( ROLE_PERMISSIONS["teacher"] + ROLE_PERMISSIONS["admin"] )) return ROLE_PERMISSIONS.get(role, []) def check_permission(role: str, permission: str) -> bool: """Check if role has specific permission.""" return permission in get_all_permissions(role) def can_access_group(user: dict, group: dict) -> bool: """Check if user can access a specific group. Admin: always True Teacher: only if group.created_by == user._id """ if user["role"] == "admin": return True return str(group.get("created_by")) == str(user["_id"])
